# React Server Components 完全指南：从入门到精通

React Server Components (RSC) 是React生态系统中的一个革命性特性，它改变了我们构建React应用的方式。本文将深入探讨RSC的工作原理、使用场景和最佳实践。

## 🚀 什么是React Server Components？

React Server Components是一种新的组件类型，它们在服务器上运行，而不是在客户端。这意味着：

- **零客户端包大小** - 不会增加JavaScript包的大小
- **直接访问后端资源** - 可以直接访问数据库、文件系统等
- **自动代码分割** - 只发送需要的组件到客户端
- **更好的SEO** - 内容在服务器端渲染

// app/posts/new/page.tsx
import { db } from '@/lib/database'
import { redirect } from 'next/navigation'
import { CreatePostForm } from '@/components/CreatePostForm'

async function createPost(formData: FormData) {
  'use server'
  
  const title = formData.get('title') as string
  const content = formData.get('content') as string
  
  if (!title || !content) {
    throw new Error('标题和内容不能为空')
  }
  
  const post = await db.post.create({
    data: {
      title,
      content,
      authorId: 'user-123' // 实际应用中从session获取
    }
  })
  
  redirect(`/posts/${post.id}`)
}

  )
}

### 2. 错误处理

```typescript
// app/posts/[id]/page.tsx
import { db } from '@/lib/database'
import { notFound } from 'next/navigation'
import { ErrorBoundary } from '@/components/ErrorBoundary'

async function getPost(id: string) {
  try {
    const post = await db.post.findUnique({
      where: { id },
      include: { author: true }
    })
    
    if (!post) {
      notFound()
    }
    
    return post
  } catch (error) {
    console.error('获取文章失败:', error)
    throw new Error('获取文章失败')
  }
}

export default async function PostPage({ params }: { params: Promise<{ id: string }> }) {
  const { id } = await params
  
  return (
    <ErrorBoundary>
      <PostContent postId={id} />
    </ErrorBoundary>
  )
}

async function PostContent({ postId }: { postId: string }) {
  const post = await getPost(postId)
  
  return (
    <article className="max-w-4xl mx-auto px-4 py-8">
      <h1 className="text-4xl font-bold mb-4">{post.title}</h1>
      <div className="text-gray-600 mb-8">
        作者：{post.author.name} | {post.createdAt.toLocaleDateString()}
      </div>
      <div className="prose max-w-none">
        {post.content}
      </div>
    </article>
  )
}
```

## 🚀 迁移策略

### 从传统React应用迁移

```typescript
// 1. 识别可以转换为Server Component的组件
// 2. 逐步迁移数据获取逻辑
// 3. 保持客户端交互组件不变

// 迁移前：客户端数据获取
'use client'
import { useEffect, useState } from 'react'

export function PostList() {
  const [posts, setPosts] = useState([])
  
  useEffect(() => {
    fetch('/api/posts')
      .then(res => res.json())
      .then(setPosts)
  }, [])
  
  return (
    <div>
      {posts.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  )
}

// 迁移后：Server Component
import { db } from '@/lib/database'

export async function PostList() {
  const posts = await db.post.findMany()
  
  return (
    <div>
      {posts.map(post => (
        <div key={post.id}>{post.title}</div>
      ))}
    </div>
  )
}
```

**优势：**
- 🚀 更快的初始加载

- 数据密集型应用
- 需要SEO的网站
