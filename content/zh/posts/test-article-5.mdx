# 前端工程化实践指南

随着前端项目的复杂化，工程化已经成为现代前端开发不可或缺的一部分。本文将全面介绍前端工程化的各个方面，帮助你构建高效、可维护的前端项目。

```javascript
  },
  testMatch: [

    '<rootDir>/src/**/*.(test|spec).(js|jsx|ts|tsx)',
  ],
};
```

### React Testing Library示例
```javascript
// src/components/__tests__/Button.test.tsx
import { render, screen, fireEvent } from '@testing-library/react';
import { Button } from '../Button';

describe('Button', () => {
  it('renders button with text', () => {
    render(<Button>Click me</Button>);
    expect(screen.getByRole('button', { name: /click me/i })).toBeInTheDocument();
  });

  it('calls onClick when clicked', () => {
    const handleClick = jest.fn();
    render(<Button onClick={handleClick}>Click me</Button>);
    
    fireEvent.click(screen.getByRole('button'));
    expect(handleClick).toHaveBeenCalledTimes(1);
  });

  it('applies disabled state correctly', () => {
    render(<Button disabled>Disabled Button</Button>);
    expect(screen.getByRole('button')).toBeDisabled();
  });
});
```

## 持续集成/持续部署

### GitHub Actions配置
```yaml
# .github/workflows/ci.yml
name: CI/CD Pipeline

on:
  push:
    branches: [ main, develop ]
  pull_request:
    branches: [ main ]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup Node.js
      uses: actions/setup-node@v3
      with:
        node-version: '18'
        cache: 'npm'
    
    - name: Install dependencies
      run: npm ci
    
    - name: Run linter
      run: npm run lint
    
    - name: Run tests
      run: npm run test:coverage
    
    - name: Build application
      run: npm run build
    
    - name: Upload coverage reports
      uses: codecov/codecov-action@v3
      with:
        file: ./coverage/lcov.info

  deploy:
    needs: test
    runs-on: ubuntu-latest
    if: github.ref == 'refs/heads/main'
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Deploy to production
      run: |
        # 部署脚本
        echo "Deploying to production..."
```

## 部署策略

### Docker配置
```dockerfile
# Dockerfile
FROM node:18-alpine as builder

WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production

COPY . .
RUN npm run build

FROM nginx:alpine
COPY --from=builder /app/dist /usr/share/nginx/html
COPY nginx.conf /etc/nginx/nginx.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

### Nginx配置
```nginx
# nginx.conf
server {
    listen 80;
    server_name localhost;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg)$ {
        expires 1y;
        add_header Cache-Control "public, immutable";
    }
}
```

## 总结

前端工程化是一个系统性的过程，涉及多个方面的优化：

- **构建优化**：选择合适的构建工具，优化打包配置
- **代码质量**：使用ESLint、Prettier等工具保证代码质量
- **自动化测试**：建立完整的测试体系，包括单元测试和E2E测试
- **CI/CD**：建立自动化部署流水线
- **性能监控**：持续监控应用性能，优化用户体验
- **文档管理**：维护完整的项目文档

通过系统性的工程化实践，我们可以：

- 提高开发效率
- 保证代码质量
- 减少bug和问题
- 提升团队协作效率
- 优化用户体验

前端工程化是一个持续改进的过程，需要根据项目特点和团队需求不断调整和优化。掌握这些工程化技能，将帮助你构建更加专业和高效的前端项目。
